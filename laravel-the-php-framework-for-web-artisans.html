Laravel: The PHP Framework For Web Artisans !

Renan

Sex 24 julho 2015
Categoria: PHP
Tags: tutorial, php, laravel
 

Entre muitos framework's para PHP, um que chamou muito minha atenção, é Laravel, quando o conheci, estava na verção 4.2, trazendo muitos aspectos que até já existiam no PHP e em outras linguagens, como a questão de migração, ORM (Object-relational mapping), linha de comando, entre outros aspectos positivos do Laravel. Atualmente esta na versão 5.1.X, com lincença MIT e seu criador Taylor Otwell é bem ativo até hoje.

Segue, a primeira parte de um tutorial sobre Laravel:

  • 0 - Requisitos básicos

    • PHP >= 5.5.9
    • OpenSSL PHP Extension
    • PDO PHP Extension
    • Mbstring PHP Extension
    • Tokenizer PHP Extension
    • Documentação

  • 1 - Instalação via composer (É a mais recomentada)

    composer global require "laravel/installer=~1.1"

    Download do composer: https://getcomposer.org/download/

  • 2 - Artisan O Artisan é o nome da linha de comando que faz interaçoes com o Laravel, é muito útil em diversas situaçoes, por agora, vamos dar um start no Laravel

    php artisan serve

  • 3 - Acessando o endereço http://localhost:8000, aparecerá a página de bem-vindo do Laravel, escrito Laravel 5.

Uma vez instalado você verá uma lista de pastas, segue a descrição para essas pastas:

app/: Diretório dos arquivos da sua aplicação.

bootstrap/: Arquivos de inicialização.

config/: Arquivos de configuração, todas as configuraçãos de sua aplicação será tratada aqui.

database/: Aqui existem basicamente duas pastas, as “migrations” armazanam arquivos que fazem as alterações na estrutura do banco (criação/auteração das tabelas, colunas, entre outros) e “seeds” são arquivos e classes que geram registros no banco de dados, para testes ou para possuir informações iniciais.

public/: Os arquivos públicos do sistema, é aqui que colocamos os arquivos estáticos (imagens, CSS e JS).

resources/: Arquivos de recursos como bibliotecas JS e CSS, arquivos com arrays de traduções de mensagens, normalmente usados numa aplicação multilinguagem e as próprias views do sistema.

storage/: Aqui fica arquivos de armazenamento de sua aplicação, desde cache, logs até sessoes.

tests/: Arquivos de testes do sistema.

vendor/: Criada pelo composer para controle e versionamento de bibliotecas.

Pessoal, essa foi a primeira postagem sobre Laravel, comentem caso tenham gostado, uma ferramenta indispensável é a Documentação desse framework, mesmo usando em produção, sempre é uma consulta muito útil.
